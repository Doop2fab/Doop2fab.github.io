Red-Black Tree

Red-Black Trees are self-balancing binary trees. It achieves this by coloring the nodes of the tree to ensure that insertions and deletions are handled in a balanced sense. The really nice benefit of a Red-Black Tree is that it provides a worst-case guarantee on insertion, deletion, and search times. This is great for real-time applications with strict requirements. On top of that by ensuring balance – they are ideal for searching and provide a significant benefit over using a Linked (or Doubly Linked) List.

Goal: For this assignment, I was tasked with creating a Red-Black Tree. My Professor wants to use the Red-Black Tree structure to model athletic roster numbers (here we assume that no double numbers are possible). He wants to know what the root node of our tree is after all insertions and which nodes are RED and which nodes are BLACK after its all said and done. He also gave us a text file that contained athletic roster numbers (integers) and my job was to read in this file and build a tree from it. He only wanted us to deal with insertions and searches for this project – no deletions.

Challenges: For all of our programming assignments, our professor made it a requirement that we put all of our data on the stack. This is an obvious challenge because it forced us to think about preventing memory leaks. On top of that, we had to program each aspect of the Red-Black Tree from scratch, all the way down to the Nodes on the tree. We were to then insert the data from the text file into the data structure whilst following the properties of Red-Black Trees.

Key Takeaways: I finished this program with a more total understanding of why programmers use Red-Black Trees and how they work down to each individual component (as I was tasked to create each one from scratch). Learning about the advantages and disadvantages, along with the specific properties of Red-Black Trees forced me to think in a different way than I ever had to think before. In addition, I had to keep my eye out for potential memory leaks, which added another layer of difficulty. I can confidently say, however, that finishing this project made me a better programmer.
